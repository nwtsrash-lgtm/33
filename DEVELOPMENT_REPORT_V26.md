## 🔧 التفاصيل التقنية

### 1. محرك الأتمتة الذكي (`engines/automation.py`)

**الهندسة:**
- `PricingRule`: كلاس يمثل قاعدة تسعير واحدة قابلة للتقييم
- `AutomationEngine`: محرك يقيّم المنتجات ضد قائمة قواعد ويتخذ قرارات
- `ScheduledSearchManager`: مدير البحث الدوري مع تتبع آخر تنفيذ
- `auto_push_decisions()`: إرسال القرارات المؤهلة إلى Make.com
- `auto_process_review_items()`: تحقق AI تلقائي لقسم المراجعة
- `log_automation_decision()` / `get_automation_log()` / `get_automation_stats()`: تسجيل واسترجاع من SQLite

**القواعد الافتراضية (3 قواعد):**
1. **خفض السعر تلقائياً**: إذا سعرنا أعلى بـ 10+ ر.س → خفض ليصبح أقل بـ 1 ر.س (حماية: لا نخسر أكثر من 15%)
2. **رفع السعر عند فرصة ربح**: إذا سعرنا أقل بـ 15+ ر.س → رفع ليصبح أقل من المنافس بـ 5 ر.س
3. **إبقاء السعر**: إذا الفرق ≤ 10 ر.س → السعر تنافسي

**حماية الأمان:**
- حد أدنى لنسبة التطابق: 90%+
- حد أقصى للخسارة: 15% من سعر التكلفة
- لا يُرسل لـ Make.com إلا بثقة ≥ 92%
- لا يبيع بسعر أقل من 1 ر.س

### 2. تحسين دقة المطابقة (`engines/engine.py`)

**المرادفات الجديدة (40+):**
- أرقام عربية: ٥٠→50، ١٠٠→100، ٢٠٠→200
- تركيزات: بريفيه→prive، ليجير→legere، ابسولو→absolue
- ماركات: توماس كوسمالا، روسيندو ماتيو، بوديسيا، كارنر، اتيليه كولون
- كلمات عطرية: عنبر→amber، زعفران→saffron، صندل→sandalwood

**تصحيح إملائي ذكي:**
- دالة `_fuzzy_correct_brand()` تستخدم rapidfuzz لمطابقة الماركات المكتوبة بأخطاء إملائية
- تُستخدم كـ fallback عند فشل المطابقة المباشرة في `extract_brand()`
- عتبة التصحيح: 85% (قابلة للتعديل)

### 3. لوحة الأتمتة في الواجهة (`app.py`)

**4 تابات:**
1. **تشغيل الأتمتة**: تقييم المنتجات + عرض القرارات + إرسال لـ Make.com + تحقق AI لقسم المراجعة
2. **قواعد التسعير**: عرض القواعد النشطة + كيفية التخصيص
3. **البحث الدوري**: مسح السوق لأهم المنتجات + تتبع الجدولة
4. **سجل القرارات**: إحصائيات + جدول بكل القرارات المتخذة

### 4. ترحيل قاعدة البيانات (`utils/db_manager.py`)

**`migrate_db_v26()` — idempotent (آمن للتشغيل المتكرر):**
- إنشاء جدول `automation_log` (سجل كل قرار أتمتة)
- إنشاء جدول `automation_settings` (حفظ الإعدادات بين الجلسات)
- إنشاء جدول `db_version` (تتبع الترحيلات)
- إضافة عمود `cost_price` لـ `our_catalog`
- إضافة عمود `auto_processed` لـ `processed_products`
- تسجيل أن الترحيل v26.0 تم تنفيذه

## ⚠️ قيود معروفة

1. **البحث في الويب**: يعتمد على DuckDuckGo + Gemini Grounding — ليس web scraping مباشر من المتاجر (يحتاج proxy/API)
2. **الجدولة الحقيقية**: Streamlit لا يدعم cron — `ScheduledSearchManager` يوفر البنية لكن التشغيل يدوي أو عبر Railway cron
3. **Vision AI**: لم يُنفَّذ بعد — مطلوب Gemini Vision API integration
4. **التنبيهات**: واتساب/بريد لم يُنفَّذ — يحتاج ربط مع Twilio أو Webhook

## 🔒 ضمان عدم فقدان البيانات

- كل تعديل على قاعدة البيانات يتم عبر `CREATE TABLE IF NOT EXISTS` و `ALTER TABLE ... ADD COLUMN` مع try/except
- لا يتم حذف أو تعديل أي جدول موجود
- سكريبت الترحيل idempotent — يمكن تشغيله أي عدد من المرات بأمان
- جميع الملفات الأصلية محفوظة — التطوير إضافي فقط
